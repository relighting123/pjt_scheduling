선택된 행동을 시뮬레이터에 전달하고, 다음 결정 시점(이벤트)까지 시뮬레이션을 진행합니다.
# env/scheduling_env.py — step() def step(self, action): bucket_idx, eqp_idx = action ppk, oper = self.decode_bucket(bucket_idx) eqp = self.eqp_ids[eqp_idx] # 1. LOT 자동 선택 lot = self.sim._auto_select_lot(ppk, oper, eqp) # 2. 장비-LOT 할당 실행 reward = self.sim.assign_ppk_oper(ppk, oper, eqp, lot) # 3. 다음 이벤트까지 시뮬레이션 진행 (DES) self.sim.advance_to_next_event() # 4. 다음 상태 관측 obs = self.sim.get_observation() terminated = self.sim.is_done() return obs, reward, terminated, False, {}

Dimension 0 — PPK_BUCKET (0–149)

공정-제품 조합 버킷 인덱스

bucket_idx = oper_idx × max_prod_count + prod_idx
= oper_idx × 10 + prod_idx
(max_oper_count=15, max_prod_count=10)
예시: oper_idx=2, prod_idx=3 → bucket_idx = 23
디코딩: oper=2, prod=3 → OPER_A, PPK_B 조합

Dimension 1 — EQP_IDX (0–9)

처리할 장비 인덱스

EQP_0 (idx=0) EQP_1 (idx=1) ... EQP_9 (idx=9)
실제 장비 수가 10개 미만이면, 초과 인덱스는 Action Mask로 자동 차단됩니다.

컴포넌트 가중치 발동 조건 계산 방식 목적
w_same_oper +2.0 직전 할당과 동일 OPER 연속 시 고정값 2.0 공정 연속성 → 장비 셋업 감소
w_same_prod +0.5 동일 제품, WIP 잔여 있을 때 고정값 0.5 (조건부) 제품 연속성 → 전환 비용 감소
w_prod_switch +0.8 이전 PPK의 WIP 소진 후 전환 시 고정값 0.8 적시 제품 전환 장려
w_completion +1.0 LOT 처리 완료 시마다 1.0 × (wf_qty / 25) 처리량 증가 장려
w_plan_hit +5.0 Plan Gap이 감소할 때마다 5.0 × Δgap 핵심: 계획 달성 장려
w_pacing +3.0 Pacing 편차가 개선될 때 3.0 × (Δdeviation↓) 균등 생산 속도 유지
w_idle_per_min −0.5 장비 유휴 시간 발생 시 −0.5 × idle_minutes 장비 가동률 극대화
w_conversion −30.0 LOT_CD 또는 TEMP 변경 시 −30.0 × n_conversions 셋업 변경 억제 (가장 강한 페널티)
w_late_finish −2.0 Soft Cutoff(05:00) 이후 완료 시 −2.0 × n_late 야간 초과 작업 억제

Pacing Shaping 상세 — _pacing_shaping_reward()

이상적인 선형 생산 속도(Ideal Line)와 실제 누적 생산량의 편차를 최소화하도록 shaped reward를 제공합니다.

# 이상적 생산량 = (elapsed / 1440) × total_plan_qty ideal = (elapsed_minutes / 1440) * total_plan_qty actual = cumulative_completed_qty deviation_prev = abs(ideal_prev - actual_prev) deviation_now = abs(ideal_now - actual_now) # 편차가 줄어들면 양의 보상, 늘어나면 음의 보상 pacing_reward = w_pacing * (deviation_prev - deviation_now)
